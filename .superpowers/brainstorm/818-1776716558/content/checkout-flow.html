Checkout-Flow für nicht eingeloggte Nutzer

Was passiert nach Klick auf „Pro abonnieren"?

A

Login-Modal → dann automatisch Checkout

Klick auf „Pro abonnieren" → Login-Modal öffnet sich mit Hinweis „Melde dich an, um fortzufahren" → Nutzer gibt E-Mail ein → Magic Link → nach Rückkehr öffnet sich der Paddle-Checkout automatisch (via URL-Parameter ?checkout=pro-monthly).

Pro

  • Nahtlos, kein zweiter Klick nötig
  • Nutzer kommt direkt zum Ziel

Con

  • Etwas komplexere Implementierung (URL-State)
B

Login-Modal → manuell auf Preisseite abonnieren

Klick auf „Pro abonnieren" → Login-Modal öffnet sich → Nutzer loggt ein → wird zurück auf /preise geleitet → sieht jetzt seinen Avatar → klickt selbst nochmal auf „Pro abonnieren" → Checkout öffnet sich.

Pro

  • Einfache Implementierung
  • Klarer, einfacher Flow

Con

  • Zweiter Klick nötig — leichte Reibung
C

Kein Login nötig — E-Mail direkt im Checkout

Checkout öffnet sich sofort. Paddle fragt im Checkout-Overlay nach E-Mail-Adresse. Nach Zahlung wird automatisch ein Account erstellt (via Webhook → Supabase). Nutzer bekommt Magic Link per E-Mail zum ersten Login.

Pro

  • Maximale Conversion — kein Vor-Login nötig
  • Weniger Schritte

Con

  • Webhook muss Account erstellen (komplex)
  • Nutzer braucht user_id für Zuordnung — ohne Login schwieriger

Empfehlung: A — beste Balance aus UX und Einfachheit. Der URL-Parameter ?checkout=pro-monthly löst den Auto-Checkout nach Login.
