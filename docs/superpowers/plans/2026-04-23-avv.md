# AVV (Auftragsverarbeitungsvertrag) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Technische Infrastruktur für den AVV nach DSGVO Art. 28: Supabase-Spalte, statische AVV-Seite, Abo-Tab-Sektion und persistenter Non-Blocking-Banner in konto.html.

**Architecture:** 
- `supabase/schema.sql` + Migrations-SQL für `accepted_avv_at` in profiles
- `frontend/avv.html` — statische, druckbare AVV-Seite (noindex, DRAFT-Text, muss von Andreas freigegeben werden)
- `frontend/konto.html` — AVV-Banner (oben in #konto-content, für paid-Nutzer die noch nicht akzeptiert haben) + AVV-Sektion im Abo-Tab
- `frontend/js/konto.js` — `_pruefeAVV()` im Init-Flow, `kontoAVVAkzeptieren()`, `_kontoZeigeAVVBanner()`

**Tech Stack:** Vanilla JS, Bootstrap 5.3.3, Supabase REST API (direct fetch wie bestehende `_pruefeAbo()`).

**⚠️ WICHTIG:** Das AVV-Dokument (`avv.html`) enthält einen ENTWURF-Text. Vor dem Scharfschalten des Banners und dem Live-Gang muss Andreas den Text rechtlich freigegeben haben. Der Banner-Code wird implementiert, aber der Banner ist **standardmäßig deaktiviert** (Variable `AVV_BANNER_AKTIV = false` in konto.js) — so kann Jens ihn manuell aktivieren sobald der Text freigegeben ist.

### Direct Fetch Pattern (wie `_pruefeAbo`)

```js
async function _pruefeAVV(userId, accessToken) {
  const res = await fetch(
    `${CONFIG.supabase.url}/rest/v1/profiles?id=eq.${userId}&select=accepted_avv_at`,
    { headers: { 'apikey': CONFIG.supabase.anonKey, 'Authorization': `Bearer ${accessToken}` } }
  );
  if (!res.ok) return;
  const rows = await res.json();
  fordifyAuth.acceptedAvvAt = rows?.[0]?.accepted_avv_at || null;
}
```

### Supabase Profile-Update (PATCH)

```js
async function kontoAVVAkzeptieren() {
  const session = _leseLocalSession();
  if (!session) return;
  const res = await fetch(
    `${CONFIG.supabase.url}/rest/v1/profiles?id=eq.${session.user.id}`,
    {
      method: 'PATCH',
      headers: {
        'apikey': CONFIG.supabase.anonKey,
        'Authorization': `Bearer ${session.access_token}`,
        'Content-Type': 'application/json',
        'Prefer': 'return=minimal'
      },
      body: JSON.stringify({ accepted_avv_at: new Date().toISOString() })
    }
  );
  if (!res.ok) { alert('Fehler beim Speichern der AVV-Akzeptanz.'); return; }
  fordifyAuth.acceptedAvvAt = new Date().toISOString();
  _kontoZeigeAVVBanner();
  kontoRendereAboTab(); // Abo-Tab aktualisieren (zeigt Akzeptanz-Datum)
}
```
